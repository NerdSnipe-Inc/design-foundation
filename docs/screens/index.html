🧾

🛍️

📊
Revenue

// Wire up an orders screen in three lines.
NavigationStack {
    DFEcommerceOrdersScreen(
        configuration: .init(
            orders: viewModel.orders,
            onSelectOrder: { order in path.append(order) },
            onRefresh: { await viewModel.reload() }
        )
    )
}

Technical contract

Same guarantees. Bigger canvas.

Every screen ships with the same standards as the primitives and blocks.

  • Typed Configuration struct per screen — no stringly-typed API, no loose dictionaries
  • All colors, typography, spacing, and radius driven by DFTheme — zero hardcoded values
  • Domain model layer per vertical — DFOrder, DFProduct, DFRevenueMetrics, etc.
  • Swift 6 strict concurrency — compiles clean with StrictConcurrency enabled
  • Action closures @MainActor-isolated — safe to call into your actor-isolated view model
  • Light and dark #Preview variants per screen, plus loading and empty state previews
  • Swift Testing suite per screen — logic testable without UI interaction
  • iOS 18 · macOS 15 · visionOS 2
  • No external dependencies beyond DesignFoundation and DesignFoundationPro

Screens depend on Blocks depend on Primitives
